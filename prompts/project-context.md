# QA Portfolio Lab — Project Context Index

This file is the entry point. Skills load only the references they need.
When you need everything, read all files in `prompts/references/`.

| Reference | Load when |
|---|---|
| `references/app-overview.md` | You need to understand the stack, tech, or ports |
| `references/auth.md` | You are writing/reviewing auth flows, protected routes, or E2E login setup |
| `references/api-contracts.md` | You are writing/reviewing API tests or checking endpoint shapes |
| `references/frontend-behaviour.md` | You are writing/reviewing E2E tests or understanding UI state |
| `references/seed-data.md` | You need to know available users, passwords, or seed kudo count |
| `references/project-structure.md` | You need to know where files go or which Playwright project to use |
| `references/pom-reference.md` | You are writing/reviewing selectors, POMs, or imports |
| `references/conventions.md` | You are writing/reviewing any test — waits, assertions, tagging, locators |
| `references/coverage-checklist.md` | You are planning coverage or need a reusable code pattern |
